    # javascript 中的 call 和 apply 实现原理

    call 和 apply 的功能是改变 this 的指向，并且立即执行使用了 call 和 apply 的函数，区别只是接受参数的方式不同。

    call func.call(thisArg,arg1,arg2,...)：第一个参数是 this 指向的对象，其它参数依次传入。

    apply func.apply(thisArg,[argsArray])：第一个参数是 this 指向的对象，第二个参数是数组或类数组。

    # 如何实现 call?

    • call是函数原型上的方法（Function.prototype.call）,因为函数都能访问到call。
    • 在call方法中获取到调用call的函数，如：func.call(obj,arg1,...),此时call中的this指向的就是func；
    • 在call方法中需要执行调用call的函数，因为使用call的函数会自动执行，如：使用func.call(otherfunc,arg1)时，func就会执行。
    • 传入call的第一个参数是调用 call 的函数中的this指向的对象，第二个或者多个参数依次传入，如：func.call(obj,arg1,arg2,arg3,...)
    • 如果第一个参数不存在，需要让调用 call 的函数中this指向window或者global。
    • 返回调用call的函数的执行结果。

    # 代码实现：

    
// 1. `call`是函数原型上的方法
Function.prototype.called = function () {
	console.log(arguments);
	// 2. 在`call`方法中获取到调用`call`的函数，如：`func.call(obj,arg1,...)`,此时`call`中的`this`指向的就是`func`；
	var constructorFunc = arguments[0];
	if (!constructorFunc) {
		constructorFunc = typeof window  !== 'undefined' ? window : global;
	}
	console.log(constructorFunc);
	// 3. 在`call`方法中需要执行调用`call`的函数，因为使用`call`的函数会自动执行.
	constructorFunc._constructor = this;
	var result = constructorFunc._constructor(arguments);
	delete  constructorFunc._constructor;
	return result;
};
